Although the DevNet git repository provides a Web-based GUI, installing a git client on your own workstation will simplify the management of your repository-based content. You can then nteract with the DevNet repository from the command line on your local workstation or from within a GUI-based repository browser, such as Atlassian SourceTree

Many modern operating systems provide a pre-installed git client; to see whether git is already installed on your OS, open a terminal window and type git --version at the command prompt; if the response provides a git version number, you're ready to use git:

$ git version 2.9.3 (Apple Git-75)

If the response does not provide a git version number, visit this page to install git. This page describes how to install git for mac, win, linux and from source. Of the Windows options, perhaps installing the GitHub client is easiest; it installs the command line tools in PowerShell as well. After you add your account info, you'll arrive at a page that asks you to choose a repository. If you are installing GitHub for Windows just to get the command-line tools that you need to complete the installation of the DevNet Learning Lab Previewer, you can skip the rest of the setup and take the following steps to launch the git command line tools in PowerShell:

  1. Click the gear icon in the upper-right corner of the window.
  2. Choose Open in Git Shell.
  3. To make sure git is working correctly, type git --version and press the Enter key. If you see a response such as git version 2.10.2.windows.1 you are now able to execute the shell-based git commands in the instructions for installing the LLP tool.

After verifying that git is installed and running on your workstation, you might want to install Atlassian SourceTree, too. (Mac only)
